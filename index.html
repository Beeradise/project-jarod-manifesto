One Mind, Three Worlds

A single learning brain, built once and proven hard in a disposable world, carried with confidence into bodies of increasing consequence — until it can finally see, and one day act, inside one it was never told about in advance.

00 — Thesis

The brain is the invariant. The body is the experiment.

Every other decision in this project follows from one architectural commitment: build one game-agnostic learning engine — NeuroCore — and port it, unmodified at its core, across substrates that get progressively closer to a body that actually has to live in a world.

The engine started in Screeps, a JavaScript colony sandbox cheap enough to break daily and disposable enough that failure is data, not damage. It was hardened there through real incidents — spawn deadlocks, poisoned learned thresholds, gradient pole-lock, off-policy control collapsing on contact with reality — until it earned 87.9% prequential forecast accuracy against a naive persistence baseline, on a live colony it had never seen tuned in advance. That number is the reason anything downstream gets built with confidence instead of hope.

From there it was ported, not rewritten, into Timberborn, a commercial C# game reached through a mod — the first time the brain had to run inside somebody else's process, on somebody else's clock, with bit-for-bit parity to its JavaScript ancestor proven in tests before a single retina was drawn. And the whole arc points at Minecraft, where the brain stops reading a game's internal memory and starts looking at a rendered screen instead — the point at which "shadow observer" finally becomes "embodied agent."

01 — The Engine

NeuroCore

A closed loop of five organs, sharing one frozen coordinate system. Every game-specific mod is a retina written against this engine — never the other way around.

The frozen coordinate frame

Every stored vector is a pure function of a label, regenerated through one seeded PRNG stream. Change the seed, the dimension, or the hashing constants after any memory exists, and every stored vector silently means something else — instant amnesia, dressed as a working program. The seed is a serialization format, not a tuning parameter, and it is the reason SeededVector(x, bigDim)[0..smallDim] == SeededVector(x, smallDim) holds exactly — growing the vocabulary is always safe; touching the frame never is.

Dimension itself is deliberately not universal — there is no single canonical hypervector size. SeededVector carries no game-specific salt, so each port picks its own geometry with zero cross-port cost: 500 on the Screeps ladder, 1000 on its private sandbox and lab servers, 12,500 soma + 1,250 world in Timberborn. A number quoted as "the" dimension is a stale or borrowed figure, not a property of the architecture.

Soma and world

Every encoding splits into two hemispheres — soma, the internal/economic state, and world, the external/ situational one. It is a distinction the manifesto borrows for itself: mechanisms that operate on stored memory read copper; mechanisms that judge, gate, or temper it read teal.

02 — The Doctrine

Laws learned the expensive way

None of these were decided in the abstract. Each one is the residue of a real incident — a colony that starved, a readout that pole-locked, a shadow predictor that looked brilliant off-policy and collapsed the moment it touched the wheel.

A learned layer hands off above the floor, never below it

The spawn arbiter's first live attempt failed in ~550 ticks: its shadow accuracy never transferred, because the deterministic floor beneath it had been silently satisfying the colony's needs the whole time it was "learning." The fix generalizes past Screeps — a learned organ governs only the surplus above a reflex floor, never the floor itself.

Every predictor is scored against a naive baseline, honestly, forever

Persistence (predict no change) and base-rate (predict the majority class) run alongside every learner. A target that can't beat them at n≥500 gets demoted or dropped — not defended. This is also how the doctrine catches its own lies: an episodic matrix that looked 87% correct in aggregate was measured at 0% in the wild until label noise and an over-wide resemblance kernel were found and fixed.

Targets must be regime-onset events with lead time, never rare-flat labels

A target that is almost always constant makes naive persistence unbeatable by construction. An early Screeps threat-forecast died this way; every predictive target shipped since is checked against the failure before it is trusted as a real "is this learning" instrument.

Persist as paged, fixed-stride binary — never a growing JSON blob

A monolithic save file that must be fully reparsed to reconstruct context does not scale, and it was measured causing real stutter. The standing target, Minecraft-region- style: file size and per-save I/O are a function of capacity, never of ticks played. The longer arc: eventually the hyperdimensional memory itself should be the context a session reads, not a JSON transcript of it.

Reason in ticks, never wall-clock

Speed multipliers and pauses make real-time a lie the simulation doesn't share. Fill rates, decay windows, and cadence are stated in ticks, and only converted to a clock when a session's actual pace is known and given.

Chunk the encoding; never bundle a mega-vector

Folding every attribute into one fixed-width bundle dilutes each attribute's marginal Hamming contribution by roughly 1/√k. Disjoint per-chunk bundles keep single-attribute change visible — and once a chunk map ships, it is an era contract: locked, never reordered.

Vigilance thresholds are derived, never scaled from an old draft

Every geometry change gets its own scratch enumeration against the real primitives — gold-pin a known vector, sweep single- and cross-chunk flips, read the actual Hamming bands. An early guess at a flora-dying threshold was uselessly wrong by an order of magnitude; thresholds are earned from data, never assumed to transfer.

Knowledge integrity binds every collaborator, Claude included

No plausible-sounding fill-in for a gap. Every claim is labeled, at least internally, as verified (checked against real code, docs, or a live source), estimated (a reasonable guess, named as one), or unknown (stopped on, researched, or asked about — never guessed past). This is not a courtesy; it has already caught real errors inside this project — a manual Ollama flag cited from a pull request that was in fact rejected, a model size that doesn't exist in any released family — before either one cost real time. Any collaborator, human or model, is held to it precisely because fluent language is not evidence of a checked fact.

03 — Three Worlds

The arc

One engine, three bodies of escalating consequence — each phase exists to earn the confidence the next one spends.

Proving ground · live

Screeps

A JavaScript colony sandbox, run three ways at once: the public ladder (main + seasonal), and two private servers — a decoupled sandbox for aggressive experiments and a disposable "lab" purpose-built to manufacture starvation events for hazard-readout training. This is where nearly every mechanism above was invented and then broken on purpose: spawn deadlocks, a wall-repair magnet that starved an economy, gradient pole-lock in two separate readouts, an on-policy spawn arbiter that failed and was rebuilt floor-protected.

prequential forecast accuracy on a live colony: 87.9%
Ported · Schema 7, final era

Timberborn

NeuroCore's first port into a compiled, closed commercial game, reached through a mod — brain (netstandard2.1, zero game references) and body (the mod's own domain retina) kept deliberately separate, with JS↔C# bit-parity proven in tests before any new capability was trusted. Seven schema eras later, the geometry is declared final: four parallel forecast targets (hazard, brownout, population decline, storage overflow), a paged binary persistence format replacing a growing JSON file, and a read-only tool that queries the memory directly instead of parsing a telemetry log — the first working rung of "read the hyperspace as context."

status: shadow observer, no game control yet
Target · vision-first

Minecraft

The point the whole arc has been pointed at: an embodied agent that navigates and acts — crafting and mining included — not a shadow forever. The deliberate departure from the first two worlds: perception stops reading the game's internal state and starts looking at the rendered screen instead, the same view a human player gets. A vision-language bridge translates pixels to semantic tokens; those tokens feed the same proven HDC/LNN/Vault substrate as any other retina. Hands — an actual action channel — are on the roadmap, deliberately not yet decided.

perception: MineCLIP + a local vision-language bridge
04 — The Support Stack

The toolchain behind the toolchain

A project run this deliberately needs its own infrastructure — hardware tuned for the dual load, and agent pipelines that build and audit the mod code the same way the doctrine demands the brain itself be measured.

Project Jarod

The bare-metal substrate: an RTX 4080 stripped to display/render cores only, zero telemetry, bypassing CUDA/cuDNN/PyTorch entirely in favor of TinyGrad compiled straight to PTX — a dual workload that has to serve real-time gaming and AI compute without either cannibalizing the other.

/triad

A three-tier build pipeline for the Timberborn C# work: one model researches and plans, a second writes and self-tests the code, a third audits the real diff against the plan and the doctrine — looping fixes back until it earns a clean pass. Every Schema-7 round shipped this way, audited before a single line was deployed.

Foreman

The local orchestrator, running entirely on Project Jarod's hardware: a resident planner/coder model, a CPU-pinned gatekeeper that never touches VRAM, and vision and summarization models swapped in only when actually needed — because a 16GB card holds exactly one heavy model at a time, and the whole design bends around that fact.

05 — Where It's Going

Open threads

Nothing here is speculative architecture — each thread is a specific, already-scoped next step.

Vision-stack bench. Characterize the chosen bridge model's real VRAM footprint, latency, and output stability; prototype the screen-capture path; design the one genuinely new encoder step this port needs — discretizing a continuous embedding into HDC-bucketable tokens.

Temporal planning, the correct way. Not new binding math — the existing state→next-state predictor, chained against itself for N-ply lookahead, shadow- scored against a naive baseline exactly like every other predictor here before it earns any authority.

Hands, deliberately deferred. Direct input injection versus a bot-API actuator — an open fork, on the roadmap, not yet decided, because perception had to be proven first.

Procedural, seed-loaded vectors. If a state's atomic and encoded vectors are pure functions of their labels, they never need to be stored — only the consolidated, learned residue does. A dimension-slicing generator would make capacity effectively unbounded, paid for in CPU instead of memory. Worth it once vision pushes the dimensionality high; not worth it in any hot path today.

The brain does not change when the body does. That is the whole argument — and the reason a colony of beavers, a hive of Screeps creeps, and, eventually, a single player named Steve can all be taught by the same mind, one honestly-measured retina at a time.
